### Objective
Целью работы является программная реализация классической игры "Угадай число" с использованием двух различных подходов: метода последовательного перебора (линейный поиск) и алгоритма бинарного поиска. Работа направлена на освоение навыков написания качественной документации (**PEP-257**), использования **аннотаций типов** и автоматизированного тестирования с помощью библиотеки `unittest`.

### Implementation
В ходе работы был реализован модуль с тремя ключевыми функциями:

1.  **`sequential_search`**: Реализует линейный перебор элементов. Количество шагов растет пропорционально размеру списка $O(n)$. Работает с любыми входными данными.
2.  **`binary_search`**: Реализует поиск методом деления диапазона пополам. Для корректной работы внутри функции производится сортировка списка `sorted(lst)`, что обеспечивает логарифмическую сложность поиска.
3.  **`guess_number`**: Универсальная функция-диспетчер, которая принимает искомое число, сформированный через `range()` список и выбранный тип алгоритма (`seq` или `bin`).



### Testing
Для проверки корректности алгоритмов реализован класс `TestGuessNumber`, включающий **13 тестовых сценариев**, покрывающих:
* Поиск в начале, середине и конце списка.
* Обработку ситуаций, когда число отсутствует (возвращается 0 шагов).
* Работу с неотсортированными списками.
* Проверку максимального количества шагов для бинарного поиска (не более 7 для диапазона в 100 элементов).
